# 阿里云部署操作文档（票务反诈 H5）

## 1. 部署目标

- 前端：Vue + Vite 静态资源上线
- 后端：Node API 服务（含 LLM 转发）
- 架构：2 台 ECS + 1 个 ALB + RDS MySQL + Redis + OSS + CDN
- 可用性：任意单台 ECS 故障时服务持续可用

## 2. 资源清单

### 2.1 计算与网络

- ECS：2 台，推荐 `2 vCPU / 4 GiB / 50 GiB`
- 镜像：Ubuntu 22.04 LTS
- 安全组开放：`22`、`80`、`443`
- ALB：1 个公网实例，后端服务器组绑定 2 台 ECS

### 2.2 数据与缓存

- RDS MySQL：`2C4G` 起步，100GB SSD
- Redis：`1C2G` 起步

### 2.3 存储与分发

- OSS：存放前端构建产物（可选）
- CDN：加速静态资源

## 3. 域名与入口规划

- 前端域名：`www.example.com`
- API 域名：`api.example.com`
- `www` 指向 CDN
- `api` 指向 ALB

## 4. ECS 初始化（两台机器都执行）

```bash
# 更新系统包索引
sudo apt-get update -y

# 安装基础工具
sudo apt-get install -y curl git nginx

# 安装 nvm
curl -fsSL https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 重新加载 shell 配置
source ~/.nvm/nvm.sh

# 安装 Node.js 20
nvm install 20
nvm use 20

# 全局安装 pm2
npm install -g pm2
```

## 5. 部署后端服务（两台机器都执行）

```bash
# 拉取代码
git clone <你的仓库地址>.git app

# 进入目录
cd app

# 安装依赖
npm ci

# 构建
npm run build
```

创建 `.env.production`（示例）：

```env
NODE_ENV=production
PORT=3000

LLM_BASE_URL=https://tokenhub.tencentmaas.com/v1
LLM_MODEL=deepseek-v4-flash
LLM_API_KEY=<你的服务端密钥>

MYSQL_HOST=<rds-endpoint>
MYSQL_PORT=3306
MYSQL_USER=<user>
MYSQL_PASSWORD=<password>
MYSQL_DATABASE=<db>

REDIS_HOST=<redis-endpoint>
REDIS_PORT=6379
REDIS_PASSWORD=<password>
```

启动服务：

```bash
# 使用 pm2 启动
pm2 start "npm run start" --name anti-fraud-api

# 保存 pm2 进程
pm2 save

# 设置开机自启
pm2 startup
```

## 6. 配置 Nginx（两台机器都执行）

写入 `/etc/nginx/sites-available/anti-fraud`：

```nginx
server {
  listen 80;
  server_name _;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }

  location /healthz {
    return 200 'ok';
    add_header Content-Type text/plain;
  }
}
```

启用并重载：

```bash
# 建立软链接
sudo ln -s /etc/nginx/sites-available/anti-fraud /etc/nginx/sites-enabled/anti-fraud

# 检查配置
sudo nginx -t

# 重载 nginx
sudo systemctl reload nginx
```

## 7. 配置 ALB

1. 创建 ALB 实例（公网）。
2. 创建服务器组，加入两台 ECS，端口 `80`。
3. 健康检查路径设置为 `/healthz`。
4. 创建监听：`80`（后续可加 `443`）。
5. 验证 ALB 地址可以访问。

## 8. 前端部署

### 方案 A：部署到 OSS + CDN（推荐）

本地或 CI 执行：

```bash
# 安装依赖
npm ci

# 构建前端
npm run build
```

将 `dist/` 上传到 OSS，CDN 回源指向 OSS Bucket。

### 方案 B：前后端同机 Nginx 托管

将 `dist/` 同步到 `/var/www/anti-fraud`，并在 Nginx 增加静态站点配置。

## 9. 证书与 HTTPS

- 在阿里云证书服务申请证书。
- `www` 证书挂 CDN。
- `api` 证书挂 ALB。

## 10. 发布流程

两台 ECS 按滚动发布：

1. 发布 `app-1`，健康检查通过。
2. 发布 `app-2`，健康检查通过。
3. 验证核心链路：Lobby -> Game -> Result -> Poster。

发布命令模板：

```bash
# 拉取最新代码
git pull

# 安装依赖
npm ci

# 构建
npm run build

# 重启进程
pm2 restart anti-fraud-api
```

## 11. 监控与告警

- ECS：CPU、内存、网络、磁盘
- ALB：QPS、5xx、后端健康状态
- 应用：接口 RT、错误率、LLM 超时率
- 数据库：连接数、慢查询、CPU
- Redis：内存、连接数、命中率

建议告警阈值：

- API 5xx > 1% 持续 5 分钟
- P95 RT > 1500ms 持续 5 分钟
- ECS CPU > 70% 持续 10 分钟

## 12. 回滚方案

- 保留上一个可用版本的 commit tag。
- 回滚步骤：

```bash
# 切换到上一版本
git checkout <上一个稳定tag或commit>

# 安装依赖
npm ci

# 构建
npm run build

# 重启服务
pm2 restart anti-fraud-api
```

两台 ECS 依次回滚，ALB 健康检查通过后结束。
