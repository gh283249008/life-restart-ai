# 小红书 Ditto 对接需求清单（开发阶段）

## 文档目的

本清单用于和小红书 Ditto 团队对齐开发输入，确保 4 页纯 H5（Lobby/Game/Result/Poster）能够顺利完成接入、联调、验收与上线。

## 一、开发阶段小红书需要提供什么

### 1. 必须提供（缺失会阻断开发）

#### 1.1 Ditto 容器能力（JSBridge）

- 是否提供 JSBridge。
- 可用方法列表（如分享、关闭页面、获取用户态、埋点上报）。
- 每个方法的参数结构、回调格式、错误码。
- 最小可运行示例。

#### 1.2 域名白名单策略

- H5 静态资源域名是否需要提前报备。
- 第三方 API 域名是否允许直连。
- 是否要求通过小红书网关/中转访问外部服务。
- 白名单生效时效和提报流程。

#### 1.3 页面挂载与路由规则

- Ditto 入口 URL 规范。
- 是否支持 `hash` 路由（本项目当前使用 hash）。
- 入口参数约定（活动 ID、渠道参数、用户标识等）。
- 刷新行为与回退行为规范。

#### 1.4 安全与审核要求

- 文案禁词、图片规范、审核约束。
- 可采集字段边界、隐私合规要求。
- 第三方模型调用合规边界。

#### 1.5 埋点规范

- 事件命名规范。
- 必填字段与字段类型。
- 上报方式（SDK、接口、网关）。
- 上报失败重试要求。

### 2. 强烈建议提供（可显著提升联调效率）

#### 2.1 联调环境

- 可调试 Ditto 测试容器。
- 测试账号及权限。
- 提审、灰度、正式发布流程。

#### 2.2 UI 与交互规范

- 安全区、导航栏、键盘弹起规则。
- 返回键与关闭页面交互标准。
- 分享样式模板规范。

#### 2.3 错误码与限流策略

- 平台侧错误码字典。
- 限流阈值与重试建议。
- 超时建议与兜底策略。

## 二、我们需要交付给小红书什么

### 1. H5 交付物

- 纯 H5 构建产物：`index.html + assets`。
- 4 页完整链路：Lobby / Game / Result / Poster。
- 可访问联调地址（测试环境、预发布环境）。

### 2. 页面与交互说明

- 4 页信息结构与跳转关系。
- 关键按钮行为。
- 返回逻辑与异常提示逻辑。

### 3. 网络与接口清单

- H5 资源域名。
- 模型接口域名与请求方式。
- 超时、重试、限流策略。
- 是否服务端中转说明。

### 4. 埋点与口径文档

- 事件列表（曝光、开始、作答、结算、海报、分享等）。
- 公共字段与业务字段定义。
- 漏斗口径与统计窗口。

### 5. 视觉素材包

- 4 页视觉稿（含移动端）。
- 立绘、海报背景、分享文案模板。
- 字体与版权声明（如涉及商用素材）。

### 6. 验收发布文档

- 联调用例与兼容机型清单。
- 已知问题与规避方案。
- 上线步骤、回滚方案、联系人。

## 三、对接优先级（建议执行顺序）

### P0（先拿到就能开始稳定联调）

1. JSBridge 文档。
2. 路由与入口规则（含 hash 支持确认）。
3. 域名白名单规则。
4. 埋点规范。

### P1（联调效率提升）

5. Ditto 联调环境与测试账号。
6. 返回与关闭规范。
7. 平台错误码与限流策略。

### P2（上线前完整收口）

8. 审核规范细则。
9. 提审与灰度流程文档。
10. 线上监控与回滚流程。

## 四、可直接发送给小红书的需求模板

```text
你好，我们这边要接入 Ditto，当前项目是 4 页纯 H5（Lobby/Game/Result/Poster）。

请协助提供以下信息：
1) Ditto JSBridge 文档（方法、参数、回调、错误码、示例）
2) H5 路由与入口规范（是否支持 hash 路由）
3) 域名白名单提报规则（H5 资源域名 + API 域名）
4) 埋点规范（事件名、字段、上报方式）
5) 联调环境与测试账号
6) 返回/关闭页面规范

我们会同步提供：H5 联调地址、页面交互说明、网络清单、埋点清单、验收计划。
```

## 五、备注

- 当前项目在纯 H5 分支推进，路由为 hash 模式，适配 Ditto 内嵌场景。
- 4 页主流程已具备基础能力，后续按小红书规范完成接口与埋点对齐。
