# 纯H5（Ditto）项目开发时间线与需求清单

## 1. 项目目标

- 在 14 天内完成 4 页纯 H5 版本（Lobby / Game / Result / Poster）。
- 完成 Ditto 内嵌适配、联调、回归测试与上线验收准备。
- 在周期内预留独立测试窗口，确保上线质量。

## 2. 项目范围

### 2.1 页面范围（必须交付）

1. Lobby 页
2. Game 页
3. Result 页
4. Poster 页

### 2.2 技术范围（必须交付）

- 纯 H5 可部署产物（`index.html + assets`）
- Hash 路由（适配内嵌刷新与回退）
- AI 对话链路（hy3 + no thinking）
- 5 轮流程、计分、结算、海报输出
- 埋点与错误观测

### 2.3 当前明确不做

- 活动规则页
- 异常降级独立页

## 3. 开发周期与里程碑

周期：`2026-05-21` 至 `2026-06-03`（14 天）

### 3.1 里程碑定义

- M1（5/24）：功能骨架完成
- M2（5/29）：业务功能完成
- M3（6/02）：测试准出
- M4（6/03）：上线就绪

## 4. 14 天时间线（每日计划）

### D1（5/21） 需求冻结与页面结构定版

- 冻结 4 页信息架构与跳转图。
- 确认 Ditto 内嵌约束（路由、返回、安全区）。
- 输出 UI 结构基线（含角色立绘位、气泡区、按钮区）。

验收标准：

- 页面结构图确认。
- 路由图确认。
- 不再新增页面。

### D2（5/22） Lobby + 路由链路稳定

- 完成 Lobby 内容与入口逻辑。
- 路由统一为 hash，入口直达能力验证。
- App 导航与页面标题统一。

验收标准：

- `/#/lobby -> /#/game` 路径稳定。
- 刷新任一路由不白屏。

### D3（5/23） Game 流程完成（不含视觉精修）

- 5 轮回合流程稳定。
- 选项判分、来源标识、失败重试打通。
- 完成 Game -> Result 数据快照写入。

验收标准：

- 连续 10 局无阻断。
- 分数与轮次一致。

### D4（5/24） Result 页完成（M1）

- 结算结果、总结、tips 展示。
- Result -> Poster 跳转。
- 再来一局、返回 Lobby 逻辑完成。

验收标准：

- 4 页主链路可跑通。
- 结算数据准确率 100%。

### D5（5/25） Poster 页完成

- 海报内容模块完成。
- 文案复制、截图分享引导完成。
- 无快照保护（直接访问容错）完成。

验收标准：

- Poster 可从 Result 100% 进入。
- 复制文案成功率 >= 95%。

### D6（5/26） Game 视觉第一轮定稿

- 漫画气泡样式收敛。
- 打字效果节奏、按钮态优化。
- 角色立绘预留区比例定稿。

验收标准：

- 关键视觉元素一致。
- 无明显错位/裁切。

### D7（5/27） Ditto 内嵌专项适配

- 安全区适配。
- 键盘弹起与可视区稳定。
- 返回行为（系统返回/页面返回）校验。

验收标准：

- iOS/Android 内嵌下主链路稳定。
- 无因键盘导致按钮不可点问题。

### D8（5/28） 埋点接入

- 曝光、开始、回合选择、结算、海报、复制等事件接入。
- 公共字段与渠道参数透传。

验收标准：

- 关键漏斗事件齐全。
- 字段完整率 >= 99%。

### D9（5/29） 可观测与稳定性（M2）

- 错误分层输出（stage/reason/status/raw）。
- AI 请求参数与重试策略收敛。
- 构建与版本标识规范化。

验收标准：

- 失败可定位可复盘。
- 功能完成定义达成。

### D10（5/30） 联调日（平台侧）

- 对接 Ditto 参数与回调。
- 白名单验证与接口可达性验证。
- 分享链路与返回链路联调。

验收标准：

- 联调问题清单形成并分级。
- P0 问题当日闭环。

### D11（5/31） 联调修复日

- 修复联调问题。
- 回归关键链路。
- 性能小优化（首屏、交互响应）。

验收标准：

- P0/P1 问题清零。
- 漏斗关键事件完整。

### D12（6/01） 全量测试窗口 Day 1

- 功能回归。
- 兼容性回归（机型、系统版本、浏览器内核）。

验收标准：

- 回归通过率 >= 95%。
- 缺陷有明确 owner 与截止日期。

### D13（6/02） 全量测试窗口 Day 2（M3）

- 重点复测。
- 漏斗与埋点数据复核。
- 出具测试报告与准出结论。

验收标准：

- P0/P1 全清。
- 回归通过率 >= 98%。

### D14（6/03） 上线验收准备（M4）

- 发布包冻结。
- 变更清单冻结。
- 回滚方案与应急联系人确认。

## 6. 需求清单（执行版）

### 6.1 功能需求

- FR-01：Lobby 可进入 Game。
- FR-02：Game 完成 5 轮回合。
- FR-03：每轮支持固定选项作答。
- FR-04：正确 +10，错误 -5，整活 0。
- FR-05：第 5 轮后进入 Result。
- FR-06：Result 可进入 Poster。
- FR-07：Poster 可复制分享文案。
- FR-08：再来一局可重新开局。

### 6.2 体验需求

- UX-01：Game 使用漫画风气泡。
- UX-02：消息展示包含打字效果。
- UX-03：左右角色立绘位置预留。
- UX-04：移动端无遮挡、可点击。

### 6.3 技术需求

- TECH-01：纯 H5 构建产物可独立部署。
- TECH-02：hash 路由适配内嵌场景。
- TECH-03：AI 调用使用腾讯 hy3 no thinking。
- TECH-04：请求超时、重试、失败观测齐全。
- TECH-05：关键数据使用前端快照保障页面间传递。

### 6.4 数据需求

- DATA-01：核心漏斗事件完整。
- DATA-02：事件字段规范统一。
- DATA-03：错误事件可溯源。

## 9. 风险与兜底

- 风险 R1：接口白名单延迟
  - 兜底：提前提报双环境域名，联调阶段先走测试域

- 风险 R2：内嵌返回行为差异
  - 兜底：统一路由回退策略，增加返回保护

- 风险 R3：模型波动导致内容失败
  - 兜底：重试、修复、检索兜底链路启用

- 风险 R4：视觉细节反复打磨压缩测试时间
  - 兜底：D12-D13 测试窗口锁定，不被挤占

## 10. 结论

本计划以“先跑通主链路，再收敛视觉与稳定性，最后集中测试准出”为核心节奏。按本时间线执行，可在 14 天内完成纯 H5 版本的开发、联调和上线验收准备。
